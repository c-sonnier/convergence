The Gaps Are Closed.

A revised editorial audit: what the first reading found, what the second draft shipped, and why one deliberate deviation still stands.

By the Convergence Review Desk Source claude.com/blog/best-practices Filed 17 April 2026 Revised 17 April 2026 · Same day Reading time 7 min
Stop Press Since this piece first ran, the maintainers shipped four of the five recommendations — parallel research dispatch, effort-level guidance, an Auto-mode section, and adaptive-thinking preambles on five skills. The fifth, a deliberate deviation on design, is now documented in the README rather than argued over. The scorecard has been redrawn accordingly.

Convergence was designed against a different model. When its fifteen skills were distilled from six open-source repos, the reference assumption was an agent that would happily spawn subagents and burn tokens in pursuit of certainty. Opus 4.7 is not that agent. It reasons more, calls tools less, and no longer accepts a fixed thinking budget — it decides how hard to think on its own.

Our first reading of the repo against Anthropic's published guidance found four silences: no mention of effort levels, no Auto-mode guidance, no adaptive-thinking prompts, and a research workflow that dispatched subagents sequentially. The recommendations at the end of that piece were small and specific. Within the same day, they shipped.

This revised edition reports on the result. The short version: the silences are gone, the scorecard reads cleaner, and the one practice Convergence still violates — batching questions in the design phase — is now legible rather than accidental. What follows is the before-and-after, a new scorecard, and the changelog that got us here.

Silence fixed. One deviation, now documented, still stands.

The Audit, Revised

Eight practices from the Opus 4.7 guidance, each measured against the skills as they shipped after revision.

Practice 01

Provide comprehensive task specifications upfront

Well-specified task descriptions — intent, constraints, acceptance criteria, file locations — outperform ambiguity spread across turns.

/convergence-design forces intent and constraints into a single 200-line artifact before any code is written; /convergence-outline adds file paths, signatures, and verification commands; /convergence-research pre-loads the relevant locations.

Strong alignment

Practice 02

Minimize user interactions & batch context

Each user turn adds reasoning overhead. Batch questions, provide full context in one pass.

The design skill inverts this: "Ask questions one at a time. Do not batch questions." The rationale is human cognitive load, not agent efficiency.

A new "Deviations from Opus 4.7 Guidance" section in the README names the exception explicitly and defends it: batched questions get shallow answers, and the cost of a bad design decision is measured in thousands of lines of code.

Undocumented deviation→Documented deviation

Practice 03

Leverage auto mode & completion notifications

Use auto mode (Shift+Tab) for long-running tasks where context is complete and you trust autonomous execution.

The README was silent on Auto mode. /convergence-implement was the canonical candidate — pre-approved outline, verification per phase, 3-failure escalation — but nothing said so.

A "Running in Auto Mode" section lists auto-safe skills (implement, review) and auto-unsafe skills (design, compound, research), with a sentence of rationale for each.

Gap→Resolved

Practice 04

Choose effort level deliberately

Default to xhigh. Reserve max for genuinely hard problems. Drop to medium or low for tightly scoped work.

No skill mentioned effort levels. Users had to guess.

Both skill tables in the README now carry an Effort column. Workflow skills default to xhigh. /convergence-architecture, /convergence-security, and /convergence-debug are flagged max. /convergence-tdd and /convergence-compound drop to medium.

Gap→Resolved

Practice 05

Prompt for adaptive thinking

Fixed thinking budgets are gone. Ask for depth — "think carefully, this is harder than it looks" — or for speed — "prioritize responding quickly."

Neither phrase appeared anywhere in the repo.

Five SKILL.md files now carry a one-line preamble immediately under the H1. debug, architecture, and security get "think carefully and step-by-step before responding." tdd and compound get "prioritize responding quickly rather than thinking deeply."

Gap→Resolved

Practice 06

State preferred response length explicitly

Opus 4.7 adapts length to complexity. Say what you want.

Research: under 500 lines. Design: under 200 lines. Research agent: under 100 lines. The instruction-budget framing in the README reinforces the discipline across every skill.

Strong alignment

Practice 07

Explicitly describe when and why tools should fire

Opus 4.7 reasons more and calls tools less by default. If you want tool usage, spell out when.

Research names its tools — "use grep, glob, and read" — and forbids RAG. Implement mandates running the verification command and reading its output. Debug, security, and review are less explicit.

Mixed — unchanged

Practice 08

Request parallel subagents explicitly

The model spawns fewer subagents by default. When fanning out across items or files, ask for parallel dispatch in so many words.

/convergence-research was the canonical fan-out case — three to seven questions, one agent per question — but its process read sequentially.

Step 2 of the research skill now reads: "Dispatch one research-agent per question in parallel — a single message with multiple Agent tool calls, not sequential dispatches. Opus 4.7 spawns fewer subagents by default, so the fan-out must be explicit."

Gap→Resolved
Four silences, one paragraph each. The repo reads current now.

What's Left

One "mixed" practice remains. /convergence-debug, /convergence-security, and /convergence-review could be more explicit about when to execute a command versus when to reason. Opus 4.7 reasons more and calls tools less by default, so this gap will probably widen over time unless addressed. It's the obvious candidate for a third edition.
